Sample Results for Commons-Math (Feb. 2016)

Compile and Tests, single-module project, 4 CPU laptop, as measured by GNU time. The value shown is the real (wall clock) time spend on the process.

Clean build from scratch

Sbt is quickest, by using all CPUs to the fullest (making the system unusable during that time). The only major change since 2016 here is that the pants buildsystem took singificantly longer than in the previously benchmarked version.

Rebuild without changes

Several buildsystems recognize there is nothing to do, surprisingly pants is not among them. A change since 2016 is that bazel now took considerable time to run again. The log shows a warning that the running server needs to be killed due to a change of start params (which did not change) so this might be a minor bug not affecting most users.

Rebuild with change

For this case I added a small no-op change to one source file.

This benchmark shows some buidsystems stop doing anything when rebuilding with a change, possibly because the change detection algorithms did not detect the change...

Rebuild first version

For this case I manually reverted the file to the state of the first build.
